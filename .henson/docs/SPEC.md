I want to build a framework for managing agents on projects that I have.
The idea is that there is a shared 'project' board - that contains tickets that can be in various states 'backlog', 'ready', 'in progress', 'review', 'done' etc. The project also needs a shared documentation folder, with a specification in - as well as any other docs (all markdown) we might want. There should also be access to the project memory for the agent in question (mostly claude). We should also be able to define a simple document that covers etiquette - always commit, always merge to main, always run tests, write few comments that sort of thing
There should be a webserver where I can go in and see all of my projects that are registered, I should be able to add tickets and make changes to documentation. Changes to docs should be monitored so that we can pull new tickets from it if changes anything.
The agents themselves should have an MCP to pull tickets/specs/memories and other docs, they should also be able to write back and update them. Each project should have a companion agent session (with a randomly generated fun name and avatar) - this agent should live alongside the project and can either do all of the work requested or offload it to a number of sub-agents (we can provide recipies for agent teams, designer, backend dev, frontend dev etc).
Projects will live on my machine in folders of my choosing - they're just normal git backed projects - agents can then be called directly from the terminal/vscode or we can provide an interface on the web-ui at some stage it would be amazing if I could just run this on a headless server and access via the web-ui.
I want a plugins system - the first plugin should be a tool for monitoring claude code usage and ensuring that we keep within session limits - even if that means running agents over multiple days. We would also want to support yolo mode in that scenario so that I can setup the board and leave the agent slowly churning through tasks never exceeding my acount limits.
